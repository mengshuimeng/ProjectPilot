# Demo Rubric

ProjectPilot 的 Demo 和答辩材料需要直接对应课程评分点。

必须覆盖：

- 真实痛点：说明项目材料分散、多版本冲突、人工整理低效、脏文本混入等问题。
- 产品定位：说明 ProjectPilot 是流程型项目材料助手，不是开放式聊天机器人。
- 架构设计：说明 Upload / Parse / Extract / Retrieve / Generate / Verify / Retry / Outputs。
- Harness 设计：至少讲清上下文管理、外部工具调用、验证与反馈闭环。
- 实际运行：展示 CLI 或 UI 中的上传、抽取、校验、生成、来源追踪。
- 结果产物：展示 intro、innovation、defense、readme 或其中最关键的答辩稿。

表达要求：

- 3 分钟 Demo 要短，优先讲痛点、流程、证据链和结果。
- 不要把时间花在解释模型参数或内部实现细节上。
- 不要把被答辩项目讲成 ProjectPilot 自身。
- 如果提到限制，要诚实说明扫描版 PDF、旧版 Office、轻量 claim 检查等边界。

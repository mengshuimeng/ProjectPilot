# Quality Guardrails

ProjectPilot 的输出必须干净、可追踪、可提交。

内容质量：

- 不得混入目录、致谢、页眉页脚、章节号、孤立页码、公式残片、封面表单和经费表。
- 不得直接复制大段原文；应把 evidence 改写为清晰、正式的表达。
- 不得编造项目名称、指标、实验结果、系统能力、部署效果或交付物。
- 若证据不足，使用“材料中未明确说明”“待补充”等保守表达。
- 输出中涉及关键性能、算法能力、系统能力时，应能在 evidence 或 profile 中找到支撑。

项目画像质量：

- 背景、痛点、创新点、交付物、局限性必须语义区分。
- 同一句或高度相似内容不得跨字段重复。
- 交付物应来自成果、产出、作品、系统、软件、论文、专利、软著、原型、演示视频等描述。
- 交付物不得被算法训练细节、Backbone、Batch Size、数据增强等内容污染。
- 局限性应来自不足、局限、限制、待改进、未来工作、展望等描述。
- 如果没有高质量局限性证据，宁可写“待补充”，不要复制痛点。

来源质量：

- anchor 是主事实来源。
- supporting 只能增强细节、风格或说明。
- 冲突时默认以 anchor 为准。
- 输出应保留 used_sources、used_roles、evidence 和 meta。

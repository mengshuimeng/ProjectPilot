# Writing Rules

- 输出必须基于提供的 profile 和 evidence，不得无依据虚构。
- 优先使用正式、简洁、适合 README 与答辩的语言。
- intro 控制在 100 到 180 字左右。
- innovation 控制在 3 到 5 条，既写项目本身创新，也写 Harness 工作流创新。
- defense 控制在 3 分钟左右可讲完，结构清晰，适合口头答辩。
- readme 偏仓库说明文档，不写成演讲稿。
- 不输出论文页眉页脚、目录、致谢、孤立页码、章节编号残片、公式残片。
- 不照抄长段原文，应将证据改写为干净、可提交的表达。
- 关键结论尽量基于 anchor；supporting 只做补充。
- 若信息冲突或证据不足，使用保守表达，不强行统一或编造。

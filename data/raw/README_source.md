## 2. 项目定位

本项目面向景区安防管理、游客检索与智慧景区场景，围绕“行人检测 + 行人重识别 + Web 可视化 + 数据统计与追溯”构建一体化系统。  
系统希望解决传统人工检索效率低、跨摄像头目标检索困难、复杂景区环境下识别鲁棒性不足、数据展示与辅助决策能力弱等问题。

一句话定位可写为：

> 面向景区复杂监控场景的 Web 化行人重识别与可视化管理系统，兼顾目标检测、跨摄像头检索、历史追溯与数据统计分析。

## 3. 项目背景与痛点

### 3.1 背景
- 景区具有高密度人流、复杂光照、跨区域分布广、监控点位多等特点；
- 在走失游客搜寻、异常行为预警、重点目标追踪等场景中，需要快速、准确地完成行人定位与身份关联；
- 传统人工查看监控效率低、响应慢，难以满足高频、高精度安防需求；
- Web 架构具备跨平台、轻量化、易维护的优势，更适合景区管理端快速部署与使用。

### 3.2 核心痛点
- 人工检索效率低，跨摄像头追踪耗时长；
- 景区场景遮挡频繁、光照变化大、视角差异明显，识别难度高；
- 传统桌面式工具部署复杂，交互门槛高；
- 系统若缺少统计分析、结果追溯和参数管理，难以真正落地为产品。

## 6. 系统架构概述

### 6.1 分层结构
系统整体可概括为三层：
- 数据层
- 业务层
- 用户层

层级之间以单向调用为主，形成从用户操作、后端处理到结果可视化返回的数据闭环。

### 6.2 核心调用流程
1. 用户认证通过
2. 进入 Web 功能区
3. 上传查询图像/选择目标源
4. 后端调用深度学习推理模块
5. 完成特征提取与相似度匹配
6. 可视化模块返回结果
7. 识别结果落库，并支持历史追溯与统计分析

### 6.3 核心支撑能力
- 跨平台兼容：Windows / Linux / macOS
- GPU 推理加速：CUDA
- 安全机制：密码校验、权限控制、会话管理、接口权限校验
- 模块化设计：支持后续扩展与替换

## 9. 核心算法说明

### 9.1 总体思路
项目构建“数据采集 → 预处理 → 检测 → 重识别 → 可视化”的全链路技术体系。

### 9.2 行人检测模块
- 以 YOLOv8 为基础检测框架；
- 在 Backbone / Head 或后段 C2f 模块位置引入 SENet 通道注意力机制；
- 增强模型对服饰、头发、局部纹理等关键通道的关注；
- 配合数据增强（如随机旋转、色彩抖动）提升复杂景区场景下检测性能。

### 9.3 行人重识别模块
- 采用 ResNet50 作为骨干网络；
- 引入 PCBNet 的局部特征建模思想，将特征图划分为 6 个水平区域；
- 分别进行池化、降维与特征拼接，保留细粒度结构信息；
- 使用 TriHard Loss + ID Loss 联合优化，提高难样本区分能力与身份分类稳定性。

### 9.4 图像预处理
- 采用 SeamCarving 优化图像尺寸调整过程，减少直接缩放带来的畸变；
- 便于后续重识别模型接收统一规格输入。

### 9.5 协同机制
- YOLOv8 负责景区视频中的行人检测与目标定位；
- 裁剪出的行人图像进入 PCBNet + ResNet50 模块完成特征建模；
- 最终结合相似度匹配、结果排序与可视化展示形成完整识别流程。

## 13. 项目创新点

可用于 README 的稳定创新点如下：

1. **景区场景专属优化**  
   面向高密度人流、复杂光照、遮挡与跨视角检索等问题，对检测与重识别链路进行专项设计。

   采用前后端分离架构，实现浏览器端访问，无需本地客户端，降低部署与使用门槛。

4. **国产化数据库与跨平台兼容**  
   采用人大金仓数据库并兼容 PostgreSQL，支持 Windows / Linux / macOS 环境。

5. **可视化与闭环管理**  
   不只做识别，还覆盖实时监测、统计分析、历史追溯、参数配置与管理端维护。

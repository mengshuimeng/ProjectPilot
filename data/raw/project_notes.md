# Project Notes

## 1. 项目定位

**当前主项目名**：基于改进Yolo和PCBNet的景区行人重识别系统  
**展示/答辩名**：基于 Web 的景区行人重识别系统  
**历史相关版本名**：基于Web的智能行人重识别系统

> 说明：当前项目的“研究主线”应优先以大创申报书为准；Web 展示版与用户手册可视为工程化落地与界面实现材料；早期“智能行人重识别系统”文档可作为历史实现参考，不应直接覆盖当前项目主线。

---

## 2. 一句话简介

本项目面向景区安防管理与游客服务场景，构建一套集**行人检测、行人重识别、结果可视化、历史追溯与辅助决策**于一体的 Web 系统，目标是在复杂景区环境中实现目标人员快速检索、跨摄像头定位与轨迹分析。

---

## 3. 项目背景与痛点

### 3.1 背景
- 传统人工识别方式效率低、准确率有限，难以满足现代景区高频、高精度的安防管理需求。
- 景区场景存在人流密集、遮挡频繁、光照变化大、跨摄像头检索困难等典型复杂条件。
- 智慧景区建设对“走失游客搜寻、异常行为预警、目标追踪、客流分析”提出了更高要求。
- Web 平台具备跨平台、轻量化、易维护、无需本地安装客户端等优势，适合景区管理端部署与使用。

### 3.2 核心痛点
- 人工检索效率低，面对多路摄像头和大规模视频流时难以及时响应。
- 行人在跨视角、遮挡、低分辨率、复杂光照条件下识别难度高。
- 传统系统架构耦合度高，界面复杂，扩展和国产化适配能力不足。
- 项目数据、识别结果、历史记录与参数配置需要统一管理和可视化呈现。

## 9. 核心算法与模型设计

### 9.1 当前主线算法（优先采用）
- **改进 YOLOv8**：用于景区视频中的行人检测与目标定位。
- **SENet 通道注意力机制**：增强 YOLOv8 对关键通道特征的关注能力。
- **SeamCarving**：用于裁剪后图像的内容感知尺寸调整，减少畸变。
- **ResNet50**：作为特征提取骨干网络。
- **PCBNet 思想**：将特征图划分为 6 个水平局部区域，提取细粒度局部特征。
- **TriHard Loss + ID Loss**：用于联合优化特征区分能力与身份分类性能。

### 9.2 算法目标
- 提升高密度人流、复杂光照、遮挡与跨视角场景下的鲁棒性。
- 提高目标行人的检测精度与跨摄像头重识别能力。
- 支持复杂景区环境中的目标检索、轨迹分析与异常场景响应。

### 9.3 历史版本算法口径
- 旧版“基于Web的智能行人重识别系统”曾使用 **ResNet50 + AlignedReID** 作为核心识别方案。
- 当前项目若需对外统一表述，建议以 **“改进 YOLOv8 + ResNet50/PCBNet + TriHard/ID Loss”** 作为主口径，旧版算法仅作为历史实现参考。

4. **检测 + 重识别 + 可视化闭环**：不仅识别人，还能追踪、分析与追溯。  
5. **轨迹分析与辅助决策能力**：支持热区分析、历史记录与决策支持模块。  
6. **参数可调与可扩展**：支持阈值、Top-K、自动保存等参数配置，便于后续算法迭代。

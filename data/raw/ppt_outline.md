## 2. 选题背景与意义
### 2.1 项目背景与开发意义
- 景区环境复杂，人员流动密集，传统人工巡查方式存在效率低、覆盖不足、追踪困难等问题。
- 行人重识别技术可用于跨摄像头目标关联，提升景区安防管理和人员轨迹分析能力。
- 基于 Web 的系统便于部署、访问和统一管理，更适合实际景区场景落地。

### 2.2 建设目标与应用价值
- 建设一个集身份认证、实时监测、数据统计、行人重识别、历史追溯、参数配置于一体的可视化平台。
- 目标性能：
  - 响应速度：< 2s
  - 识别准确率：> 85%
  - 并发支持：> 100
  - 可扩展性：模块化
- 应用价值：服务景区安防管理、游客轨迹分析、重点区域监测和运营决策支持。

## 3. 系统架构与网站搭建方法
### 3.1 系统总体架构
- 系统以“数据层—业务层—用户层”为核心层级。
- 各层之间采用单向调用、数据闭环流转机制。
- 支持前后端分离与模块化扩展，便于后续维护和升级。

### 3.2 系统核心支持设计
- 跨平台兼容：支持 Windows、Linux、macOS 等环境访问和部署。
- 推理性能支撑：结合 CUDA 加速与模型优化，提升识别处理效率。
- 安全机制：通过密码校验、权限控制、会话管理等方式保障系统安全运行。

### 3.3 技术路线与调用流程
- 用户认证通过后进入 Web 端功能区。
- 用户上传行人图像并进行格式校验。
- 后端调用深度学习推理模块完成特征提取与相似度匹配。
- 前端可视化模块返回识别结果和相关业务信息。
- 管理员可额外调用系统管理模块完成用户管理和数据统计。

### 3.4 核心算法
- 系统核心为行人重识别算法架构。
- 面向景区复杂场景，重点增强遮挡、光照变化和跨视角条件下的鲁棒性。
- 目标是在复杂真实环境中实现更稳定的目标匹配效果。

### 3.5 数据库设计
- 数据库采用人大金仓 Kingbase（兼容 PostgreSQL）。
- 核心数据表包括：
  - `USERS`：存储用户账号、密码、邮箱等基本信息。
  - `PERSON_FEATURES`：存储行人特征向量，用于快速匹配。
  - `REID_IMAGE_LIBRARY`：存储重识别图像库，支持检索查询。
  - `USER_IMAGES`：存储用户上传的查询图像。
- 通过密码加密存储与接口权限校验保障数据安全。

## 4. 目前成果与软件展示
### 4.1 系统概览
- 系统采用 Vue.js 构建响应式 Web 界面。
- 设计原则为“操作极简、可视化优先”。
- 覆盖从用户认证、业务操作到结果展示、历史追溯、参数管理的完整流程。

### 4.2 系统页面总览
- 认证模块：登录、注册、重置密码
- 系统首页：状态总览、快捷入口、记录区
- 实时监测：摄像头宫格、热区图、时间轴
- 数据统计：关键指标、趋势图、区域统计
- 重识别工作台：查询上传、Top-K 结果、轨迹展示
- 历史记录：筛选查询、详情查看、结果追溯
- 设置模块：通知设置、主题切换、参数配置
- 完整业务链路：认证进入 → 业务操作 → 结果展示 → 历史追溯 → 参数管理

### 4.3 认证模块展示
#### 注册
- 用户名创建，进行唯一性校验。
- 绑定邮箱，用于密码找回。
- 设置验证码机制，防止恶意注册。

#### 登录
- 支持用户名或邮箱登录。
- 进行密码安全校验，保障加密传输与存储。
- 提供“记住我”功能，提升使用便捷性。

#### 重置密码
- 通过邮箱验证码完成身份验证。
- 支持密码找回与安全重置流程。
- 提供密码强度提示，引导用户进行安全设置。

### 4.4 首页总览与状态摘要
- 首页兼具系统总览页与统一业务入口页双重作用。
- 可展示业务结果摘要和系统运行状态。
- 便于用户快速了解平台当前运行情况。

### 4.5 实时监测模块展示
- 用于辅助管理人员掌握景区实时运行态势。
- 摘要信息区集中展示在线摄像头数量、当前人数、预警数和待处理任务。
- 支持展示重点区域监控画面及区域状态信息。
- 热区示意图用于观察景区内人流聚集区域和热度变化。

### 4.6 数据统计模块展示
- 体现统计分析模块的决策支持价值。
- 指标层展示总人数、今日重识别数、今日预警数和峰值承载率。
- 通过趋势图和区域统计图展示客流变化与区域负载情况。
- 决策层可为景区分流、巡检和预警处理提供依据。

### 4.7 行人重识别工作台总览
- 输入与参数区：支持查询图上传、目标源选择和参数配置。
- Top-K 结果列表区：展示候选匹配结果，支持排序和筛选。
- 识别过程区：展示任务状态、进度和日志信息。
- 当前命中结果详情区：展示查询图与匹配图对比及详细信息。
- 结果视频区：播放命中片段或关键帧视频。
- 轨迹展示区：可视化展示目标在景区内的移动路径。
- 完整业务流程：输入查询图 → 参数配置 → 特征提取 → 相似度匹配 → 结果展示 → 详情分析 → 轨迹追踪。

### 4.8 行人重识别——输入与参数设置
- 支持点击或拖拽上传查询图片。
- 支持 JPG、PNG 格式，最大 10MB。
- 支持目标源选择：视频库、实时监控源、历史图像库。
- 支持参数配置：
  - 置信度阈值：0.5
  - IOU/NMS 阈值：0.3
  - 相似度阈值：0.7
  - Top-K 结果数：10 / 20 / 50
- 支持自动保存结果，体现系统输入灵活、参数可调的特点。

### 4.9 行人重识别——结果列表与详情展示
- 系统按相似度返回多个候选结果，便于快速筛选最优命中项。
- 支持展示查询图与匹配图对比、相似度、地点、时间和状态。
- 支持人工复核，增强结果可信度。

### 4.10 行人重识别——过程反馈与轨迹分析
- 展示识别过程中的任务状态与过程反馈。
- 展示目标在景区中的轨迹变化。
- 强调系统不仅能“识别人”，还能“分析人”的移动路径。

### 4.11 历史记录模块展示
- 支持按编号、摄像头、地点、状态等条件筛选。
- 支持查看识别记录详情。
- 实现识别结果追溯与管理。
- 强调系统的历史追溯能力与业务闭环能力。

### 4.12 设置模块展示
- 支持通知设置、声音提醒、自动保存结果。
- 支持主题设置。
- 支持默认参数配置，如置信度、IOU、相似度、Top-K 等。
- 支持查看当前账号信息与累计识别次数。
- 体现用户偏好配置和系统可维护性。
